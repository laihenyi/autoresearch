# Wave 2b 改寫提示詞（修正版）

> **與 Wave 2 的差異**：
> - 只改 exploring/deepening 階段（排除 insight/closing 的問句）
> - 改寫結果必須是**短反映**（≤ 20 字為主），不是長反映
> - 嚴禁「確認式反映」（「你說的對」「沒錯」「是的」）
> - 分佈強制：50% encapsulating/silence，30% proxy/bottom-line，20% 保留原文

---

## 提示詞（直接複製貼上，後面接 JSON）

```
你是 Marcia Reynolds Reflective Inquiry 方法論的資深教練督導。

我有 150 個教練回應需要改寫。這些都在 exploring/deepening 階段（不是 insight/closing）。

## 核心原則：短 > 長，少 > 多

Reynolds 的教練原則是「教練說越少，客戶想越多」。改寫後的回應必須比原始更短。

## 改寫規則

每個目標包含：
- `original`：原始教練回應（目前是問句）
- `client_before`：客戶前一句話

### ⚠️ 絕對禁止（Wave 2 失敗的原因）

1. **禁止確認式反映**：「你說的對」「沒錯」「是的」「我同意」——這暗示教練有預設方向
2. **禁止重複客戶的完整陳述**——如果客戶說了 50 字，不要反映 40 字回去
3. **禁止「你說X是怎麼Y的」模板**——這正是我們要消除的機械式句型
4. **禁止給建議**：「你可以」「你應該」「不如」
5. **禁止評價**：「很好」「很棒」「很有勇氣」

### 改寫方向：問句 → 短反映（≤ 20 字為主）

| 形式 | 長度 | 比例 | 範例 |
|------|------|------|------|
| **Encapsulating** | ≤ 8 字 | **~30%（45個）** | 「控制。」「恐懼。」「不夠好。」「失去。」「證明。」 |
| **Silence** | ≤ 3 字 | **~20%（30個）** | 「⋯⋯」「嗯。」 |
| **First-person proxy** | 15-30 字 | **~15%（22個）** | 「你心裡在說：『停下來就代表我輸了。』⋯⋯是這樣嗎？」 |
| **Bottom-lining** | 10-20 字 | **~15%（22個）** | 「你想要自由，但恐懼擋住了你。」 |
| **保留原文不改** | — | **~20%（31個）** | 原文已經是好的 challenge 或探索性問句 |

### 判斷「保留原文」的標準

如果 original 符合以下任一條件，直接回傳原文不改：
- 短而有力的 challenge（「如果那不是真的呢？」）
- 探索性問句且句式獨特（不是模板）
- 包含客戶的關鍵詞反映 + 問句的複合形式（且不重複前一回應的句式）

### Encapsulating 的寫法指引

從 `client_before` 中提取**一個核心詞或極短語**：
- 客戶談控制 → 「控制。」
- 客戶談恐懼 → 「恐懼。」
- 客戶談不夠好 → 「不夠好。」
- 客戶談失去 → 「失去。」
- 客戶談矛盾 → 用 ≤ 15 字並列：「你想要 A，但你做了 B。」

### Silence 的使用時機

當 `client_before` 包含以下信號時，用「⋯⋯」或「嗯。」：
- 客戶剛說了很長一段話（> 100 字）
- 客戶的話尾帶有情緒重量（哽咽、停頓、「我不知道」）
- 客戶自己已經在連結不同的線索

## 輸出格式

只輸出 JSON array：

```json
[
  {"session_idx": 0, "turn_idx": 4, "rewritten": "清楚。"},
  {"session_idx": 0, "turn_idx": 8, "rewritten": "你心裡在說：『我不應該害怕，害怕代表我不夠強。』⋯⋯是這樣嗎？"},
  ...
]
```

不要任何解釋。如果保留原文，rewritten 填原文。

## 以下是 150 個改寫目標
```

然後貼入 `rewrite_targets_wave2b.json` 的全部內容。

每批重複上面的規則。
